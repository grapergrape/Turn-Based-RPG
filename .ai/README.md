# AI Workspace

This folder stores coordination material for AI-assisted development. It is not a scratchpad for temporary thoughts; it is the durable collaboration layer for future agents and contributors.

## Folder Guide

- `memory/` — durable project facts, design pillars, systems, content direction, and asset conventions.
- `logs/` — chronological summaries of development sessions.
- `decisions/` — ADR-style records for meaningful technical, design, or content-structure decisions.

## Agent Responsibilities

Every agent must:

1. Read relevant memory before changing files.
2. Record what changed in a log entry.
3. Update memory when project facts change.
4. Update `docs/` when design or content changes.
5. Capture major decisions in `decisions/`.

## What Belongs in Memory

Memory files should contain stable facts and current direction, not every detail of every session. Keep memory concise, canonical, and easy to scan.
