# Project Memory

Memory files contain durable information future agents should know before making changes.

## Files

- `project.md` — project identity, pillars, scope, current status, and unresolved high-level questions.
- `story.md` — canonical narrative, setting, factions, characters, and quest direction.
- `systems.md` — gameplay systems, combat rules, progression, inventory, and interaction models.
- `assets.md` — art, animation, audio, UI, and asset naming conventions.

## Update Rules

Update memory when a fact becomes durable or when project direction changes. Do not copy entire session logs here; summarize only the canonical outcome.
